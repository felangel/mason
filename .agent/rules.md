# User Rules and Guidelines

These are the rules that Antigravity should follow for every request in this project:

1. **Documentation Persistence**: Every new functionality, feature, or major change requested MUST be reflected in the relevant `README.md` files (both the root `README.md` and the package-specific `README.md` in `packages/masonex/`).
2. **Language**: Technical explanations (like the file generation process) and code should be in English, but conversation with the user can be in Spanish.
3. **Consistency**: Ensure any new features align with the existing `masonex` and `mustachex` architecture patterns.
